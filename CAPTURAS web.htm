1. Permisos para Google Maps y Ubicación (AndroidManifest.xml).

Estos permisos son la "llave de entrada" para acceder a los servicios de ubicación del dispositivo. Sin ACCESS_FINE_LOCATION la app no podría obtener coordenadas precisas del GPS, y sin INTERNET no podría comunicarse con los servidores de Google Maps. Son los cimientos legales que Android exige para apps que usan geolocalización.

  1. MapaActivity.java - onMapReady() (Muestra ubicación y ruta en mapa).

 

Este método es el "director de orquesta" del mapa. Coordina la colocación de marcadores (usuario y distribuidora), invoca el trazado de rutas y ajusta la vista para mostrar ambos puntos. Es donde la teoría de la ubicación se convierte en visualización tangible para el usuario.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


  1. MenuActivity.java - calcularDistanciaPorRuta() (Calcula ruta y costo de despacho).

 

Actúa como el "navegante inteligente" de la app. No se conforma con distancia en línea recta, sino que consulta la API Directions de Google para obtener la ruta real por calles, calculando distancias precisas que reflejan la realidad del transporte.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


  1. MenuActivity.java - calcularCostoDespacho() (Calcula valor del despacho por distancia).

 

Es el "algoritmo de negocio" que transforma kilómetros en dinero. Implementa la lógica comercial: envíos gratis para compras sobre $50k y en un radio de 20 km, tarifas preferenciales para compras medianas, y tarifa completa para compras pequeñas, reflejando una estrategia de precios realista.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


  1. PagosActivity.java - onCreate() (Pasarela de pagos ficticia).

 

Simula ser el "cajero virtual" de la tienda. Aunque no procesa pagos reales, ofrece una experiencia completa de checkout con múltiples métodos de pago, mensajes de confirmación y transición fluida, manteniendo la ilusión de una transacción real.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


Compilación

El proceso de compilación inicia con Gradle realizando una validación exhaustiva de todas las dependencias críticas: Google Play Services para mapas y ubicación, Firebase Auth para la autenticación de usuarios, Google Maps SDK para la visualización cartográfica, y OkHttp para las llamadas a la API de Directions. Simultáneamente, se verifica la correcta configuración de la API Key de Google Maps en el AndroidManifest y la declaración adecuada de los permisos de ubicación. La integración con Google Maps API se consolida mediante la clave específica, configurando los servicios de ubicación y el SupportMapFragment en los layouts, mientras que Firebase Authentication se vincula validando las reglas de seguridad y integrando el SDK correspondiente. Durante la compilación de la lógica de negocio, se procesan los algoritmos centrales que incluyen el cálculo de rutas mediante Directions API, el sistema de costos de despacho por distancia, la gestión del estado del carrito de compras y la simulación de la pasarela de pagos. El compilador luego optimiza y empaqueta todos los recursos gráficos, el código Java convertido en bytecode DEX, y las configuraciones de permisos y actividades, culminando en la generación de un APK que integra armónicamente un frontend con interfaz intuitiva, servicios de geolocalización con mapas interactivos, autenticación empresarial y APIs externas para ofrecer una experiencia completa de e-commerce geo-contextualizada.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


Ejecución

 

  1. LoginActivity.java es la puerta de entrada donde los usuarios se autentican con Firebase mediante email y contraseña.

 

 

 

 

 

  1. CompraActivity.java funciona como el supermercado virtual, permitiendo seleccionar productos y gestionar el carrito de compras.

 

 

 

 

 

 

 

  1. MenuActivity.java actúa como el calculista de envíos, obteniendo la ubicación GPS del usuario y determinando el costo de despacho según la distancia real.

 

 

 

 

 

 

 

  1. MapaActivity.java se convierte en el visualizador de rutas, mostrando en Google Maps la ubicación del usuario, la distribuidora y el trayecto de entrega.

 

 

 

 

 

 

 

  1. PagoActivity.java simula la caja registradora virtual, donde se confirma el pago total y se selecciona un método de pago para completar la transacción.
